# 市场温度页面（SMA20 宽度）

## Goal

新增一个「市场温度」功能页面，展示 **全市场** 与 **各行业** 的个股 MA20 站上率（价格站上 20 日均线的个股占比），让用户一眼判断当前市场冷热及行业分化。

## 背景与关键约束（第一性原理）

- 项目现为 **ETF 级** 数据（`etfs.json`/`themes.json`），**无任何个股行情**（`stocks_spot.json` 为空），自建全市场个股宽度管线成本极高（复权/停牌/退市/新股）。
- 采用 **复用外部现成宽度** 路径：`dapanyuntu.com`（大盘云图）已按个股计算好 **二级行业 MA20 站上率**，每日 16:00 更新。这是拿到可信个股宽度的最短路径。
- **硬约束（已与用户确认）**：
  - 数据源只到 **行业** 粒度，不给个股明细 → **主题维度本期不做**（无法精确到主题成分股）。
  - **「全市场」= 各行业站上率的等权均值**（数据源不提供成分股数，无法个股加权）→ 页面须明示口径。
  - `value == 0` 表示 **无数据**，非真实 0%，聚合与展示时过滤。

## Requirements

### 数据维度
- 维度：**全市场**（等权均值）+ **行业**。
- 行业粒度 **可切换**：默认 **26 个一级行业**，可下钻 **86 个二级行业**。
- 历史范围：**31 个交易日**（数据源单次返回窗口，约 6 周）。

### 数据管线
- 新增 `dapanyuntu` 数据源，带 `Referer`/`User-Agent` 头拉取 `industry_ma20_analysis_page`（WebFetch 直连会 403）。
- 计算并产出快照 `market_temperature.json`：同时含 **二级原始值、一级聚合值、全市场均值序列**。
- 86 二级 → 26 一级映射表移植进 backend（来源：`market-breadth-heatmap-skill` 的 `industry-mapping`）。
- 接入现有 `cn-refresh` 每日 cron，随现有流程写 `data/latest/` 并归档进 `data/snapshots/<date>/`，重建 snapshots-index。

### 页面（全量三块）
- **① 全市场温度计**：当日站上率大数字（如 30.4%）+ 冷热配色 + 31 日迷你趋势线。
- **② 行业排行**：当日按站上率排序的条形榜，一级/二级切换（默认一级）。
- **③ 历史热力图**：行业（行）× 交易日（列）颜色矩阵，随①②的一级/二级切换联动。
- 落地为新 tab/route，与「主题轮动 / 跨市雷达」并列。

## Acceptance Criteria

- [ ] 后端 `dapanyuntu` provider 能带头拉取并解析 `{data, dates, industries}`，403/超时/空数据有降级与日志。
- [ ] 快照 `market_temperature.json` 结构含 `dates` / `market` / `industries_l1` / `industries_l2`，`0` 值已过滤。
- [ ] 全市场当日数字与 dapanyuntu 当日各行业均值一致（±0.1）。
- [ ] 一级 26 行业、二级 86 行业均正确展示；一级为二级按映射表等权均值聚合。
- [ ] 前端新 tab 可访问；温度计、行业排行、历史热力图三块渲染正确，31 列对齐。
- [ ] 一级/二级切换时三块联动。
- [ ] 页面明示「全市场=行业等权均值」口径说明。
- [ ] 数据随 `cn-refresh` cron 每日更新并归档；snapshots-index 含新文件。
- [ ] 后端单测（provider 解析、聚合、全市场均值、0 值过滤）+ 前端组件测试通过。

- 个股级自建宽度管线。
- 成分股加权的「全市场」口径。
- 站上其它均线（MA5/MA60）——本期只做 MA20。
