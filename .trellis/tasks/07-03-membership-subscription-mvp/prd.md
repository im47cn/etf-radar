# 会员订阅 MVP（爱发电订阅闭环 + 自选盯盘）

## Goal

为 ETF Radar 引入第一版会员订阅，验证真实付费意愿并产生收入。定位「活跃散户 × 效率/全景/沉淀」，**卖工具与信息，不荐股**。首版聚焦最小可收款闭环：登录 → 订阅 → 会员权益（自选盯盘）。

## 背景与约束

- 前端为 GitHub Pages 静态站，**无自有后端服务器**；已接入 Supabase（OAuth 登录 + Portfolio）。
- 收款通道：**爱发电（afdian）** 做 MVP（个人可注册、免营业执照/免 ICP 备案、支持微信/支付宝、有 webhook）。跑通后二期再迁微信官方支付。
- **合规红线**：中国境内对个股/ETF 给出明确买卖建议并收费 = 证券投资咨询业务，需持牌。本产品**只卖分析工具与客观信息，用户自行决策**。全站文案只描述客观状态（转强/转弱/温度升降），**禁止操作动词**（买入/加仓/清仓），并展示免责声明「仅供参考，非投资建议」。
- 现有分析数据在公开 repo（`data/latest`、`data/snapshots`）中，人人可抓取——故付费价值**不建立在锁现有公开数据上**，而建立在「每用户私有能力（自选）+ 服务端门控」。

## MVP 范围（首版必做）

1. **登录**：复用现有 Supabase OAuth，无需新建认证。
2. **订阅闭环**：
   - 站内「升级」页为已登录用户生成一次性**绑定码**。
   - 引导用户前往爱发电订阅对应档位，并在订单**留言中填写绑定码**。
   - `afdian-webhook` Edge Function 接收 afdian 回调、验签、按留言中的绑定码匹配到 Supabase user，写入/更新 `subscriptions`。
   - 前端 `useSubscription` 读取订阅状态并解锁会员功能。
3. **自选盯盘**：会员可对主题 / A股 ETF 加入自选，数据存 `watchlist`（RLS 门控），前端提供自选视图（筛选出自选项的当前状态）。
4. **升级页 / 会员中心**：展示权益、定价、绑定码、爱发电引导链接、当前订阅状态（未订阅 / 生效中 + 到期日）。

## 明确排除（二期，不在本任务）

- 邮件「每日变化摘要」推送（Resend + pg_cron）。
- 全量历史快照回看与 CSV 导出（涉及 pipeline 改造 / 公私数据拆分）。
- 微信服务号模板消息推送、微信/支付宝官方支付迁移。
- 优惠券、试用期、退款自动化。

## Acceptance Criteria

- [ ] 未登录用户访问升级页 → 引导登录；已登录未订阅用户 → 看到定价 + 绑定码 + 爱发电引导。
- [ ] 已登录用户在升级页可获得唯一绑定码（同一用户重复进入复用未使用的有效码）。
- [ ] 爱发电订阅成功后，webhook 能在留言含正确绑定码时把该用户 `subscriptions.status` 置为 `active` 并写入 `expires_at`。
- [ ] webhook 验签失败或绑定码无效/缺失时不激活，且记录可排查（不静默吞错）。
- [ ] `useSubscription` 能正确返回 `loading / member / non-member` 三态；会员到期后回落为 non-member。
- [ ] 非会员无法读写他人 `watchlist`、无法通过前端绕过拿到自选写权限（RLS 强制）。
- [ ] 会员可增删自选主题/ETF，自选视图正确展示所选项的当前状态。
- [ ] 全站会员相关文案无操作动词，含免责声明。
- [ ] 前后端既有测试全绿；新增数据层与 hook 有单测覆盖。

## Notes

- afdian webhook 不含 Supabase user_id，故采用「绑定码写入订单留言」方案完成账号打通，避免额外调用 afdian query API。
- 会员真实价值数据必须服务端强制门控（RLS / Edge Function）；前端 `useSubscription` 仅用于 UX，不作为安全边界。
