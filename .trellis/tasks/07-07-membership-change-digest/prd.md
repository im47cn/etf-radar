# 会员每日变化摘要邮件推送（二期）

## Goal

为会员提供**每日一封的自选变化摘要邮件**，作为会员订阅的**留存引擎**——让用户不必每天盯盘也能第一时间知道自己关注的主题/ETF 的客观状态变化。卖「省时间/及时」，**不卖预测**（合规 + 规避信号弱软肋）。

## 背景与约束

- 关联 [会员订阅 MVP](../archive/2026-07/07-03-membership-subscription-mvp/)：自选盯盘（`watchlist`）+ 订阅（`subscriptions`）已上线。本任务是其留存补全。
- 数据来自 Python 数据管线的每日 snapshot（`data/snapshots/<date>/`，含 themes/etfs 强度/宽度、市场温度）。
- 前端静态站 + Supabase serverless；推送计算放 GitHub Actions（EOD 归档后），复用 snapshot 本地数据算时间差。
- **合规红线**：纯客观状态描述，**零操作动词**（买入/加仓/卖出/建议/看涨），邮件含免责声明「仅供参考，非投资建议」+ 退订链接。
- **信号弱前提**（见项目记忆 `signal-backtest-findings`）：内容只报「状态变化」不报「会涨跌」，价值定位为「省去每日盯盘」。

## 触发事件（A+C+D，按标的聚合降噪）

> **B（宽度跨档）已在 research 阶段剔除**：theme/ETF 无自身宽度维度，`market_temperature` 仅全市场/巨潮门类/中类级，且无 theme→行业映射（tags 与门类命名对齐率 <10%）。降级需引人工映射表，违背「不引新魔数」，故本期不做。详见 research/q1。

- **A 象限迁移**：自选项在轮动象限（长期×短期强度，中线 50 四象限）跨象限。
- **C 全市场温度档位切换**：全局事件，所有会员都收，置顶。温度分档边界 30/50/70（真源 `frontend/src/lib/breadthColor.ts`，Python 侧同值移植 + 单测钉边界）。`market_temperature` 缺失日安全跳过 C。
- **D 强度评分跨 50 中线**：自选项双轨强度评分（`strength.composite`）上穿/下穿 50。

**降噪原则（关键）**：**按标的聚合，不按触发器聚合**。同一自选项当天同时触发 A/D 时，合并为**一行**，取最显著变化描述 + 强度数值。阈值一律复用现有分档边界，**不引入新魔数**。

## 投递规则

- 每天最多 1 封；**当天该用户自选无任何变化 → 不发**（保持信号高）。
- 仅发给**生效中的会员**（`subscriptions` active 且未过期）。
- 触发时机：CN EOD 归档完成后（现有 `cn-eod-archive` cron，18:00 BJT 后）。
- 渠道：邮件（Resend）。用户邮箱取自 Supabase auth。微信服务号模板消息为更后续（本任务不做）。
- 退订：每封含一键退订链接（写 `notify_prefs.enabled=false`）。

## 明确排除（不在本任务）

- 微信服务号模板消息推送。
- 盘中实时推送 / 多封/天。
- 自定义推送阈值、用户级偏好细调（除退订开关外）。
- 历史回看、CSV 导出（另属二期其它任务）。

## Acceptance Criteria

- [ ] EOD 后自动运行；当日无自选变化的用户不收到邮件。
- [ ] 会员的自选项发生 A/C/D 任一变化时，收到一封摘要邮件；同一标的多触发（A+D）合并为一行。
- [ ] 全市场温度档位切换时，所有生效会员的邮件置顶该条；`market_temperature` 缺失日跳过 C 不报错。
- [ ] 非会员（无订阅/已过期）不收到邮件。
- [ ] 邮件文案零操作动词，含免责声明与可用的退订链接；点击退订后不再收到。
- [ ] 变化判定基于今日 vs 上一交易日 snapshot（按实际存在目录回溯，非目录名减一天），阈值复用现有分档边界（温度 30/50/70），无新魔数。
- [ ] 计算/发送对无自选、无会员、数据缺失（昨日 snapshot 不存在）等边界安全降级，不报错中断。
- [ ] 新增逻辑有单测（变化 diff、聚合降噪、会员过滤、退订）；既有测试全绿。

## Notes

- 邮箱来源：Supabase `auth.users.email`；退订状态存 `notify_prefs`（MVP 已设计该表）。
