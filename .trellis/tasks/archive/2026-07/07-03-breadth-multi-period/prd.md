# 个股宽度多周期(MA20/60/120)自建管线

## Goal

把「市场温度」从依赖 dapanyuntu 的**单一 MA20、行业级**，升级为**自建、个股级、多周期(MA20/60/120)**的宽度指标：全市场与各行业的个股价格站上 SMA20/60/120 的比例，提供短/中/长期视角切换。dapanyuntu 降级为 MA20 对账校验源。

## 背景与关键事实（研究已确认）

- dapanyuntu **只有 MA20**（端点/参数全试过，MA60/120 拿不到）→ 多周期只能自建。
- 项目已有 `data/stocks/close_series.json`：**全市场 5531 只个股收盘价矩阵**，由 `stocks_history_pipeline.py`（新浪 `ak.stock_zh_a_daily`）生成，`--days` 控制深度，**当前 75 交易日**。
- 加深 75→150 **几乎零成本**：新浪返回整支全历史、只是 `tail(days)`，调用次数不变，存储 ~5MB。
- ⚠️ 陷阱：daily 增量 `stocks_daily_pipeline.py` 的 `WINDOW_DAYS=75` 会把矩阵**截回 75 天**，加深后必须同步提到 150。
- **股票→行业映射项目里不存在**，是最大新增件。**M0 spike 已定源（见 research/m0-industry-mapping-spike.md）**：
  - 东财成分接口 **CI 实测 0/5 全挂**（RemoteDisconnected），证伪。
  - 申万成分接口 akshare 1.18.64 有 bug、官网 SSL 被墙，不可用。
  - **选定巨潮 `stock_industry_change_cninfo`（逐股，本机稳定）**：响应自带完整层级（门类/大类/中类/次类），**映射与分类体系一次到手**；一级=行业大类、二级=行业中类，taxonomy 从数据自然长出，无需手维护映射表。
  - 逐股 5531 次调用复用现有 `stocks_history_pipeline` 的 ThreadPoolExecutor 并发骨架（该骨架 CI 已验证可稳定跑 5531 次 akshare 调用）；月级低频 + 持久缓存 + 断点续跑。

## Requirements

### 数据与口径
- 周期：**MA20 / MA60 / MA120** 三档，全局切换、默认 MA20。
- 维度：**全市场** + **行业**（26 一级 / 86 二级，沿用现有折叠树）。
- **口径升级（本功能核心价值）**：因为拿到个股，
  - 「全市场」= **全体个股**站上率（真·个股占比，**不再是行业等权均值近似**）。
  - 「行业」= 该行业**成分股**站上率。
  - 站上判定：`收盘价 > SMA_n`（前复权，与 close_series 一致）。
  - 无效样本过滤：停牌/缺失(null)、上市不足 n 个交易日的新股（SMA 无法计算）→ 不计入分母。
- 行业分类：**巨潮体系**（一级=行业大类，二级=行业中类，来自 M0 spike 决策）。**注意：页面行业名将从东财体系变为巨潮体系，用户会看到行业重新组织**（可接受，已确认）。

### 数据管线
- 历史深度加到 **150 交易日**（MA120 需 ≥120）：backfill days=150 + daily `WINDOW_DAYS=150`。
- 新增**股票→东财二级行业**映射数据（缓存 JSON，低频刷新，成分变动慢）。
- 新增**自建宽度计算**：读 close_series + 映射，逐股算 SMA20/60/120、判定站上、按行业+全市场聚合，每周期一套序列。
- **dapanyuntu 保留**为 MA20 对账，但**降级为仅全市场**（巨潮 taxonomy 与东财不一致，行业级无法 apples-to-apples）：自算全市场 MA20 vs dapanyuntu 全市场 MA20 均值，产出偏差 + 超阈告警日志（QC）。
- 接入现有 cron 调度。

### 前端
- 全局 **MA20/60/120** 切换器，三块（温度计/排行/热力图）联动换周期。
- 沿用现有折叠热力图与自带切换的排行；口径文案更新为「个股级真实占比」。
- 快照 schema 升级（多周期），前端优雅处理新旧。

## Acceptance Criteria

- [ ] 历史深度稳定维持 150 交易日（backfill + daily window 同步），MA120 可算出 ≥1 个有效点。
- [ ] 股票→巨潮行业映射数据生成且覆盖率合理（≥95% 的 close_series 个股有大类/中类归属；未归属计入日志）。
- [ ] 自建宽度快照含 MA20/60/120 三套，每套有 全市场序列 + 一级 + 二级；停牌/新股正确排除出分母。
- [ ] 全市场口径为**个股占比**（可人工抽样核验：某日站上数/有效样本数）。
- [ ] dapanyuntu 对账（仅全市场）：自算全市场 MA20 与 dapanyuntu 全市场均值的偏差有量化输出；偏差在合理阈值内（方法学微差可接受，阈值在 design 定）。
- [ ] 前端 MA20/60/120 全局切换三块联动；MA120 在历史不足时优雅提示而非报错。
- [ ] 口径文案更新为个股级真实占比。
- [ ] 后端单测（SMA 计算、站上判定、新股/停牌过滤、行业聚合、全市场占比、对账偏差）+ 前端组件测试通过。
- [ ] 数据随 cron 每日更新；schema 升级后归档/索引正确。

## 非目标（YAGNI）

- ST 含/不含开关（个股层面理论可做，但本期不做，避免 scope 膨胀）。
- MA5/MA10/MA250 等其它周期。
- 主题维度宽度。
- 东财/申万行业（M0 已否，走巨潮）。
- 复权方式可选（固定前复权，与 close_series 对齐）。
- 行业级 dapanyuntu 对账（taxonomy 不同源，仅全市场对账）。

## 关键风险（M0 已定源, 剩余）

1. **巨潮逐股 5531 次调用在 CI 的稳定性/耗时**：复用现有并发骨架缓解；月级 + 缓存 + 断点续跑；失败保留旧 map 不阻断。
2. **巨潮多分类标准多行**：`stock_industry_change_cninfo` 每股返回多标准多行（巨潮/中证/旧版），需筛"最新巨潮标准"一行；实现须明确筛选规则。
3. **自算全市场 MA20 与 dapanyuntu 的方法学差异**：成分范围/复权差异，对账阈值据实定。
4. **taxonomy 变更用户感知**：行业名从东财变巨潮，页面行业重排（已确认可接受）。
